# СтройПроект v0.22.0 «Clarity» — резюме изменений

**Дата:** 05.07.2026 · **База:** v0.21.0 «Measured» · **Архив:** `STR.RAG V0.22.0.zip`

Релиз по замечаниям с ЖИВОГО прогона (скриншоты пользователя) + наведение
порядка в папках. Все решения проверены dex-панелью из 6 моделей (deepseek,
groq, cerebras, mistral, gemma, qwen — все «ОК», замечания учтены).

## Интерфейс (главное — «по моделям запутано всё»)

1. **Сайдбар упрощён.** Убрано дублирование настроек моделей: в сайдбаре теперь
   только — провайдер/модель ПО УМОЛЧАНИЮ, ключи API (свёртка), статус локальных
   моделей одной строкой и read-only сводка «кто чем отвечает» (М1/М3/М4 →
   провайдер → модель → ключ). Настройка ИИ конкретного модуля — только в его
   вкладке; заголовок блока сразу показывает «🤖 ИИ этого модуля: DeepSeek ·
   deepseek-chat».
2. **Граф связей (М3) информативнее:** размер узла = число связей; ПМООС
   (целевой раздел) выделен цветом; тултипы «даёт данные: N · использует: M»;
   легенда прямо на графе; стабилизированная раскладка.

## Диагностика проблем с живого прогона

3. **«Индексация не запускается»** — теперь видно ПОЧЕМУ: запуск фонового
   процесса обёрнут в try/except (точная причина в статусе и журнале — частые:
   антивирус блокирует python, битый venv), команда запуска пишется в журнал,
   toast с pid при старте, кнопка **«🩺 Проверить окружение»** (файлы найдены /
   дочерний python запускается / модуль импортируется / состояние пишется).
   Путь журнала больше не искажается (markdown съедал `\.` — казалось, что путь
   без слеша; это было только отображение).
4. **«Почему есть нераспознанные»** — файлы вида «Раздел 3.1.2 …» несут только
   НОМЕР раздела по ПП-87, а классификатор знал лишь аббревиатуры. Теперь номер
   распознаётся (точный «5.4»→ОВиК; нет точного — подъём к родителю
   3.1.1→3.1→3). Проверено на именах со скриншота: ИГДИ/ИГИ/ИГМИ/ИЭИ и
   «Раздел N том M» распознаются. Оставшиеся UNKNOWN добираются кнопкой
   «🤖 Распознать разделы с ИИ».

## Структура папок STR.RAG (по образцу ЭКОДОК)

- `releases\` — ТОЛЬКО архивы версий + РЕЗЮМЕ + LATEST.txt (распакованные
  копии удалены; код живёт в git-репозиториях).
- `Раунды замечаний\Раунд 4…8` — вместо «N замечания — копия — копия…».
- Корень: `_GH_MRAG_` (канонический код), `STR.RAG` (репо GitHub), `releases`,
  `Раунды замечаний`, ТЗ.

## Проверка

py_compile всех изменённых файлов; headless AppTest (хаб + М2/М3/М4) — без
исключений; классификатор — 12/12 тестовых имён; dex-панель 6 голосов — все
решения подтверждены. Живой прогон на вашем ПК: обновите приложение до v0.22.0,
нажмите «▶ Индексировать» — если снова не стартует, теперь будет видна точная
причина (статус/журнал/«Проверить окружение»).
