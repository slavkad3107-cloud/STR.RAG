# СтройПроект v0.23.0 «Chunking» — резюме изменений

**Дата:** 06.07.2026 · **База:** v0.22.0 «Clarity» · **Архив:** `STR.RAG V0.23.0.zip`

Семантический чанкинг как **OPT-IN** — по вердикту dex-дебата («мерить A/B, а не
внедрять вслепую»): всё готово к сравнению, но **по умолчанию поведение НЕ
меняется** (режим `char`, как раньше).

## Что добавлено

1. **Семантический чанкер** (`chunk_text_semantic`): режет текст не по символам,
   а по СМЫСЛОВЫМ границам — нумерация пунктов НПА («5.4.», «3.1.2)»),
   «п./пункт/раздел/таблица/приложение», заголовки капсом, абзацы — собирая
   цельные положения до ~512 токенов bge-m3. Таблицы не трогаются (их собирает
   merge_tables при поиске). Сверх-крупные блоки дробятся по предложениям.
   Покрыт юнит-тестами: границы, отсутствие потери текста, лимиты, цельность
   таблиц, детерминизм id.
2. **Config** `chunking`: `mode: "char"|"semantic"` (по умолчанию `char`),
   `target_tokens: 512`, `chars_per_token: 3.2`.
3. **Переиндексация «с нуля»**: кнопка «♻ Переиндексировать заново» в М2 и CLI
   `--reindex` — удаляют коллекцию Qdrant и обрабатывают все файлы заново (нужно
   при смене режима: дедупликация по doc_sha иначе пропустила бы файлы, и новый
   чанкинг не применился бы). В М2 показывается активный режим нарезки.

## Как провести A/B на вашем ПК (5 минут)

```bat
cd _GH_MRAG_
:: 0) индекс и принятые ответы уже есть (М2 + М4). Собрать эталон:
python scripts\eval_golden.py build --project "ИМЯ"
:: 1) BASELINE (текущий char-режим):
python scripts\eval_golden.py run   --project "ИМЯ"
:: 2) включить семантический режим: в %USERPROFILE%\.pmoos-rag\config.yaml
::    chunking.mode: semantic   (или создать config.yaml с этим ключом)
:: 3) М2 -> «♻ Переиндексировать заново» -> «Да, с нуля» (дождаться)
:: 4) ПОСЛЕ:
python scripts\eval_golden.py run   --project "ИМЯ"
```
Сравнить recall@k и MRR (история — в `%USERPROFILE%\.pmoos-rag\eval_history.jsonl`).
Если семантический режим ВЫШЕ — оставляем его; если нет — вернуть `mode: char`
и переиндексировать. Так решение по чанкингу принимается по цифрам, а не на глаз.

## Дальше (по вердикту дебата, после этого A/B)

- Если чанкинг дал прирост → **Contextual Retrieval** (LLM дописывает 1–2 строки
  контекста к чанку) — усиливает эффект коротких вырезок из больших СанПиН.
- «Масштаб» (Qdrant int8/mmap/HNSW) — только при симптомах нехватки RAM на 100+
  проектах.
