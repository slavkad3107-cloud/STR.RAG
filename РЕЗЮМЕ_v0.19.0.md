# ПМООС-RAG v0.19.0 «Tuned» — резюме изменений

**Дата:** 30.06.2026
**Базовая версия:** v0.18.0 «Split» (включает оптимизации v0.17.0 «Faster»).
**Состав:** `V0.19.0/` (исходники) + `STR.RAG V0.19.0.zip` (внутр. папка `V0.19.0`) + это РЕЗЮМЕ.

Пакет оптимизаций по итогам ПО-МОДУЛЬНОГО аудита через DEX (Claude + cerebras,
кросс-LLM). Без изменения пользовательского поведения; приоритет — качество и стабильность.

## Что нового (по модулям)

- **М4 — персистентный BM25-индекс.** Корпус (ids/texts/payloads/tokens)
  кэшируется на диск (`<проект>/bm25_corpus.pkl`); инвалидация по числу точек в
  Qdrant. Больше НЕ `scroll`-им всю коллекцию при каждом запуске пайплайна — на
  повторных прогонах BM25 читается с диска. + пул кандидатов перед реранком 40→60
  (лучше recall; fp16-реранкер компенсирует стоимость).
- **М3 — кэш графа зависимостей** по mtime `graph.json`. `downstream`/
  `explain_cascade` больше не читают граф с диска на каждое замечание
  (≈150 → 1 чтение за прогон М4). При отсутствии файла граф строится на лету и
  не кэшируется (без «залипания»).
- **М1 — дисковый кэш OCR** (sqlite по sha страницы+язык, общий между процессами,
  FIFO-лимит). Повторная индексация сканов не гоняет pytesseract заново.
- **М6 — валидация кодов ЗВ** перед выгрузкой УПРЗА: пустые/нераспознанные («—»),
  необычный формат, дубли. Статус — в колонке `Статус` файла `uprza_vybrosy.csv`
  и сводкой в `ЗАДАНИЕ_УПРЗА.txt`.

**Осознанно НЕ менялось:** парсинг таблиц PDF (pdfplumber) — смена движка таблиц
рискованна для числовых данных ООС (приоритет качества). Тяжёлые внешние
зависимости (Unstructured.io, igraph, Neo4j, camelot+Java), предложенные
советником, отклонены — против офлайн/малозависимого принципа.

## Проверка

py_compile всех изменённых файлов; импорт без циклов; unit-смоук каждой правки
(персистентный BM25: scroll один раз / диск-хит без scroll / инвалидация по count;
кэш графа: 1 загрузка на 3 вызова + инвалидация по mtime; OCR-кэш: hit без
pytesseract + FIFO-лимит; валидация ЗВ: пустые/формат/дубли) — всё зелёное;
headless AppTest хаба и модульных приложений — без исключений.
Адверсариальный мультиагентный ревью (19 агентов): 15 находок, 3 low — исправлены
(кэш графа при отсутствии файла; FIFO-лимит OCR-кэша; «—» как «нет кода» в валидации).

**На вашем ПК (RTX 3070 Ti) перед боевым использованием:** живой прогон М4 на
наборе замечаний — проверить, что персистентный BM25 даёт те же/лучше результаты
и ускоряет повторные запуски; качество ответов не просело.
