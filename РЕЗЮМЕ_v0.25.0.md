# СтройПроект v0.25.0 «Onboarding» — резюме изменений

**Дата:** 06.07.2026 · **База:** v0.24.0 «Guardrails» · **Архив:** `STR.RAG V0.25.0.zip`

Улучшение **удобства и информативности интерфейса** для инженера-эколога (не
программиста). Метод: сначала снята фактическая карта интерфейса headless-тестом
(`streamlit.testing` — все заголовки, подписи, тултипы, кнопки, экспандеры), затем
эта карта разобрана мульти-LLM панелью **/dex** (5 моделей: deepseek/cerebras/
gemma/qwen/groq) по 7 аспектам UX. Внедрены пункты, по которым модели сошлись как
«высокая ценность / низкий риск» и которых ещё не было в приложении. Все правки
безопасны, БЕЗ переиндексации, без изменения бизнес-логики.

## Что улучшено

1. **Сквозной индикатор прохождения (главное).** На самих вкладках теперь виден
   статус ключевых шагов: `✅ М1` · `⏳ М2` · `○ М4` (✅ готово · ◑ частично ·
   ⏳ идёт · ⏸ пауза · ⚠ ошибка · ○ не начато). Под заголовком проекта — строка
   «что делать дальше» по фактическому состоянию (напр. «▶ Дальше — М2: постройте
   RAG-базу») плюс краткая сводка (файлов распознано, проиндексировано, ответов).
   Убирает ощущение потерянности между шестью модулями. Только чтение JSON —
   дёшево, без сети и моделей.
2. **Режим нарезки чанков — прямо в интерфейсе (М2).** Раньше сменить можно было
   только правкой `config.yaml` — барьер для не-программиста. Теперь выпадающий
   список «По символам / По смыслу»; выбор сохраняется в конфиг, с явным
   предупреждением «вступит в силу после ♻ Переиндексировать заново».
3. **М1 — заметное предупреждение о нераспознанных файлах.** Если у файлов не
   определён раздел, вверху выводится `⚠`-плашка с тремя способами исправить
   (переименовать по образцу «Раздел N. …» / задать вручную / кнопка ИИ) и
   причиной. Кнопка «🤖 Распознать разделы с ИИ» становится акцентной (primary),
   а блок ручного уточнения раскрывается автоматически и показывает счётчик.
4. **Акцентная кнопка запуска индексации (М2).** «▶ Индексировать» теперь primary —
   очевидное главное действие модуля.
5. **Человеческое пояснение к М2.** Добавлена строка простыми словами: «программа
   читает документы и строит поисковую память, по которой ИИ находит места для
   ответов» — снимает страх перед терминами RAG/чанки/эмбеддинги.

## Проверка (тест)

- Снятие карты интерфейса и повторная проверка — `streamlit.testing.v1.AppTest`.
- Комплексный прогон: **хаб + все 6 отдельных модульных приложений** —
  7/7 рендерятся без исключений и без дублей ключей.
- Целевые смоук-тесты: индикатор «что дальше» считается верно (М1 частично, М2
  готово → «Дальше М4»); `⚠`-плашка М1 появляется при нераспознанном файле;
  селектор режима нарезки присутствует; конфиг `chunking.mode` корректно
  сохраняется (round-trip в data-dir, ключи API не пишутся); баннер версии
  показывает `v0.25.0 «Onboarding»`.

## Что НЕ трогали (уже было и работает)

Прогресс-бар индексации, статусы-бейджи и диагностика запуска в М2; плашка «⚠ без
опоры на источники» в М4 — эти вещи модели предлагали «добавить», но они уже есть
с v0.22–v0.24. Дублировать не стали.

## Что дальше (роадмап без изменений)

A/B семантического чанкинга (теперь переключается прямо в М2), по результату —
contextual retrieval; «масштаб» (Qdrant int8) — при симптомах.
