# СтройПроект v0.21.0 «Measured» — резюме изменений

**Дата:** 04.07.2026
**Базовая версия:** v0.20.0 «Rebrand».
**Состав:** `V0.21.0/` (исходники) + `MGRAG V0.21.0.zip` + это РЕЗЮМЕ.

Релиз сформирован по итогам **мульти-LLM дебата** (плагин dex v0.5.3, режим
auto=debate, 3 раунда + вердикт судьи с кросс-семейным аудитом; панель:
DeepSeek, Groq/Llama-3.3, Cerebras, Mistral, Gemma, Qwen; cohere/gemini
отпадали по гео/квоте). Главный вердикт: **«сначала измеритель — потом
улучшения качества»**.

## Что нового

1. **Golden-set eval harness** (`scripts/eval_golden.py`) — единогласный №1 дебата:
   - `build --project "ИМЯ"` — собирает эталоны из ПРИНЯТЫХ ответов (замечание,
     принятый ответ, файлы-источники) → `golden.jsonl`;
   - `run --project "ИМЯ" --golden golden.jsonl [--k 8] [--expansion]` —
     retrieval-метрики: recall@k и MRR по файлам-источникам;
   - `judge --project "ИМЯ" --golden golden.jsonl` — LLM-сравнение текущих
     ответов с эталонными (балл 0–10, слабые ответы списком).
   История замеров: `<данные>\eval_history.jsonl` (версия и метрики).
2. **JSON-повтор в ПАКЕТНОМ пути М4** — битый JSON в батче получает один
   batch-повтор «верни ТОЛЬКО JSON» без кэша (раньше — пустой ответ).
3. **Векторный few-shot памяти экспертизы** — похожие принятые ответы ищутся
   эмбеддингами bge-m3 (косинус; ловит перефразированные замечания), настройки
   `memory.semantic` (вкл) и `memory.min_sim` (0.45); при любой ошибке —
   прежний лексический поиск. Эмбеддинги идут через общий дисковый кэш.
4. **Fallback-цепочка LLM** — `ai.fallback_provider` (например `ollama` или
   `openai`): один повтор через резерв при сбое основного. По умолчанию ВЫКЛ.

## Решения дебата (для истории)

- **Смысловой чанкинг ~512 токенов** (№2 дебата, польза 5) — в СЛЕДУЮЩЕЙ версии,
  строго после снятия baseline на harness (мерить A/B, не внедрять вслепую).
  Факт: 1200 символов русского ≈ 250–400 токенов bge-m3 — в лимит 1024 влезает;
  выигрыш от резки по пунктам НПА, а не от размера.
- **Qdrant int8/mmap/HNSW** — отложено 2:1 (нет симптомов нехватки RAM; потеря
  recall 1–2% недопустима для цитат экспертизы). Вернуться при росте базы.
- **Contextual Retrieval** (LLM-контекст к чанкам) — отложено 2:1 (дорого:
  LLM-вызов на каждый чанк; сначала подтвердить эффект чанкинга на eval).
- LibreOffice для .doc — отложено (не влияет на качество).

## Рекомендованный цикл работы с harness (на вашем ПК)

```bat
cd V0.21.0
python scripts\eval_golden.py build --project "ИМЯ"      && rem один раз
python scripts\eval_golden.py run   --project "ИМЯ"      && rem baseline
rem ... после любого изменения системы: run/judge снова и сравнить
```
