# СтройПроект v0.29.0 «Complete» — резюме изменений

**Дата:** 07.07.2026 · **База:** v0.28.0 «Swift» · **Архив:** `STR.RAG V0.29.0.zip`

Закрытие ВСЕХ отложенных пунктов из ревью v0.26–v0.28, безопасных без
golden-set замера. Качество ответов не тронуто.

## Новое

1. **Вкладка «М7 · Качество»** — измеримость для владельца без терминала:
   - «① Собрать эталон» одной кнопкой (эталон = ваши принятые ответы из М4;
     пишется `golden.jsonl` проекта) — дёшево, только чтение answers.json;
   - готовые команды для замера поиска (recall@k, MRR) и ИИ-судьи —
     копируются из интерфейса (тяжёлые шаги осознанно НЕ в Streamlit:
     грузят модели/платные вызовы LLM);
   - **история замеров** (eval_history.jsonl) таблицей — тренд качества между
     версиями и настройками (например, ДО/ПОСЛЕ смены режима нарезки в М2).
   Это закрывает главный пробел измеримости: теперь A/B семантического чанкинга
   делается по шагам «М7 ① → команда ② → М2 смена режима + reindex → команда ② →
   сравнить строки в М7».

2. **Near-dup фильтр стал version-aware** (опция `retrieval.dedup_near`): из
   группы почти-одинаковых чанков разных редакций тома выживает АКТУАЛЬНАЯ
   (корр/изм/v3 > исходная — по маркерам версии в имени файла), а не первая
   по силе поискового сигнала. Раньше в ответ могла попасть устаревшая
   формулировка. Позиция группы в пуле сохраняется (реранкер дальше рескорит).

## Оптимизация/архитектура

3. **Кэш content_signature по doc_sha** — simhash+minhash (~1 с чистого Python
   на файл) не пересчитывается при reindex неизменённого контента.
4. **Сетевые функции вынесены из интерфейса** (`pmoos/ingest/remarks_fetch.py`):
   скачивание файла замечаний по ссылке/Google Drive — теперь в ядре, hub.py —
   тонкая обёртка (алиасы сохранены для совместимости).

## Проверка

19 юнит-тестов; 7 смоук-проверок нового (gdrive-разбор, алиасы hub, version-aware
выживший — «корр» побеждает «старый», позиция группы сохранена, подпись с doc_sha);
AppTest хаб (включая М7) + 6 модульных приложений — 7/7 без исключений.

## Осталось только то, что требует замера или данных

- Включение семантического чанкинга по умолчанию — после A/B через М7 (теперь
  доступно без программиста).
- Contextual retrieval — после подтверждения чанкинга (нужна переиндексация).
- Извлечение таблиц PDF связными строками; статусы редакций НПА (нужен слой
  данных); lookahead для латинских «no»/«co» (нужен golden-set: риск потерять
  легитимные NO/CO в таблицах выбросов).
